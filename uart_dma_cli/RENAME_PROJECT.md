# Rename Guide

This template is designed to be renamed from the `.ioc` level and then regenerated by STM32CubeMX or STM32CubeIDE.

## What to Rename
- `uart_dma_cli.ioc`

Example:
- `uart_dma_cli.ioc` -> `motor_controller.ioc`

## Recommended Rename Procedure
1. Duplicate the template repository into a new repository.
2. Rename `uart_dma_cli.ioc` to your new project name.
3. Open the renamed `.ioc` file in STM32CubeMX or STM32CubeIDE.
4. Update the project name in CubeMX project settings.
5. Regenerate the STM32CubeIDE project files.
6. Open the regenerated project and verify it builds correctly.

## Why This Template Does Not Include `.project`
- IDE metadata files store fixed project identity and workspace-oriented settings.
- Reusing them directly from a template often causes name conflicts or import issues.
- Regenerating from `.ioc` ensures the new project gets clean metadata for its own name.

## What Usually Stays Unchanged
- `Core/`
- `Drivers/`
- `STM32F103RBTX_FLASH.ld`
- HAL/CMSIS source tree

## What Gets Regenerated
- `.project`
- `.cproject`
- `.mxproject`
- `.settings/`
- launch files

If the regenerated STM32CubeIDE project opens under the new name and builds normally, the rename is complete.
